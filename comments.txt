1. Таблица brands
brand_id: SERIAL = INTEGER + автоинкремент. Оптимально для первичного ключа (максимум 2.1 млрд значений).

brand_name: VARCHAR(50) — названия брендов обычно короткие (Audi, BMW). 50 символов хватит с запасом.

origin_country: VARCHAR(50) — страны также имеют короткие названия.
2. Таблица colors (Цвета)
color_id: SERIAL — стандартный выбор для ID.

color_name: VARCHAR(30) — названия цветов обычно короткие ("red", "dark blue"). UNIQUE исключает дубликаты.
3. Таблица customers (Покупатели)
customer_name: VARCHAR(100) — с запасом для длинных имен 

phone: VARCHAR(30) — номера могут быть длинными .
4. Таблица car_models (Модели автомобилей)
model_name: VARCHAR(50) — хватает даже для длинных названий ("Mercedes-Benz S-Class").

base_gasoline_consumption: NUMERIC(5,2) — хранит числа вида 12.34.

5 цифр всего, 2 после запятой → максимум 999.99 (достаточно для расхода).

NULL разрешен, т.к. у электромобилей (Tesla) нет расхода топлива.
5. Таблица cars (Автомобили)
car_id: INTEGER (не SERIAL), т.к. ID берутся из исходного файла.

price: NUMERIC(12,2) — хранит цены до 999,999,999.99 (хватит даже для дорогих авто).

discount: NUMERIC(5,2) — скидка обычно до 100% (100.00), но с запасом.

sale_date: DATE — оптимально для дат (без времени).
6.Таблица car_colors (Связь автомобилей и цветов)
car_id и color_id: INTEGER — совместимы с внешними ключами.

Составной PRIMARY KEY — гарантирует уникальность связки (машина не может быть одного цвета дважды).
